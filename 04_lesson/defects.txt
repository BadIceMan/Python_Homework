ОТЧЁТ О ДЕФЕКТАХ КЛАССА StringUtils

----------------------------------------------------------------------
ID: BUG-01
Компонент: StringUtils.contains(string, symbol)
Приоритет: Критический (Critical)
Описание: Метод полностью падает со значением TypeError при передаче None в качестве искомого символа.
Шаги воспроизведения:
  1. Вызвать метод `utils.contains("SkyPro", None)`
Ожидаемый результат: Метод должен вернуть `False`, так как тип данных некорректен, либо обработать ошибку.
Фактический результат: Программа аварийно завершается с ошибкой `TypeError: argument should be integer or bytes-like object, not 'NoneType'`.
Причина: Конструкция `try-except` обрабатывает только ошибку `ValueError` (когда символ не найден), но полностью игнорирует некорректные типы данных.

ID: BUG-02
Компонент: StringUtils.delete_symbol(string, symbol)
Приоритет: Критический (Critical)
Описание: Метод падает с ошибкой AttributeError при передаче None вместо обрабатываемой строки.
Шаги воспроизведения:
  1. Вызвать метод `utils.delete_symbol(None, "k")`
Ожидаемый результат: Возврат пустой строки или генерация понятного исключения.
Фактический результат: Аварийное падение `AttributeError: 'NoneType' object has no attribute 'index'`.
Причина: Метод вызывает `self.contains(string, symbol)` без предварительной проверки входного параметра `string` на равенство `None`.
